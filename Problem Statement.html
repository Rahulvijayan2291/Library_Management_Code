Library Management API – Spring Boot

In this challenge, you are tasked with building a REST API service for managing a library using Spring Boot. The API should enable CRUD (Create, Read, Update, Delete) operations for books in the library. You will need to implement functionality to add, update, retrieve, and delete book information. The team has provided detailed requirements for the API endpoints, including request formats, response codes, and query structures.

Book Attributes:

  • id: the unique ID of the book [int]
  • ISBN: The unique International Standard Book Number of the book [String]
  • Title: The title of the book [String]
  • Author: The author(s) of the book [String]
  • Description: A brief description of the book [String]
  • Genre: The genre or category of the book [String]
  • PublicationYear: The year the book was published [int]
  • CopiesAvailable: The number of copies of the book available in the library [int]
  • Publisher: The publisher of the book [String]
  • CoverImageUrl: The URL of the book's cover image [String]

Sample Book JSON:

{
  "ISBN": "9780061120084",
  "title": "To Kill a Mockingbird",
  "author": "Harper Lee",
  "description": "A novel set in the American South during the 1930s, focusing on the trial of a black man accused of raping a white woman.",

  "publisher": "J. B. Lippincott & Co.",
  "coverImageUrl": "https://via.placeholder.com/150"
}

API Endpoints:

  1. Adding a new book:

    • Endpoint: POST /book/add
    • Request Body: JSON object containing book details
    • Response Code: 201 if successful, 400 if data validation fails

  2. Updating book information:

    • Endpoint: PUT /book/{id}
    • Request Body: JSON object containing updated book details
    • Response Code: 200 if successful, 400 if ID validation fails

  3. Retrieving all books:

    • Endpoint: GET /book/get
    • Response Code: 200 if successful, 400 if no books exist

  4. Deleting a book by ID:

    • Endpoint: DELETE /book/{id}
    • Response Code: 200 if successful, 400 if ID validation fails

Instructions:

  • Run the application using mvn spring-boot:run.
  • Test the application using mvn clean test.
  • If encountering a "port already in use" error, run fuser -k 8080/tcp in the terminal or change the port in application.properties.
